# repasoEstructura
Sistema de inventario en C++ aplicando estructuras de datos fundamentales para gestión y optimización de información.
Descripción

Este proyecto consiste en el desarrollo de una aplicación para la administración de un inventario utilizando distintas estructuras de datos fundamentales en C++.

El objetivo es aplicar conceptos teóricos en un entorno práctico, permitiendo gestionar productos, clientes y operaciones de forma eficiente.

Objetivos
Implementar diferentes estructuras de datos.
Comprender su funcionamiento y aplicaciones.
Simular un sistema real de inventario.
Mejorar la eficiencia en búsqueda, inserción y eliminación de datos.
Estructuras de Datos Implementadas
1. Arreglos

Se utilizan para almacenar un listado inicial de productos.

Características:

Almacenamiento estático.
Acceso rápido a los elementos.
Ordenamiento de productos por:
Nombre
Precio
2. Listas Enlazadas

Permiten manejar el inventario de forma dinámica.

Funciones:

Agregar productos.
Eliminar productos.
Buscar productos por código.
3. Pilas (Stack)

Se utiliza para llevar un historial de operaciones realizadas.

Características:

Máximo de 10 operaciones.
Estructura LIFO (Last In, First Out).
Permite deshacer la última acción.
 4. Colas (Queue)

Simula la atención de clientes.

Características:

Estructura FIFO (First In, First Out).
Atención en orden de llegada.
Gestión de pedidos de clientes.
 5. Árbol Binario de Búsqueda (ABB)

Permite almacenar productos de forma ordenada y eficiente.

Funciones:

Inserción de productos.
Búsqueda rápida por código.
Recorrido inorden para mostrar productos ordenados.

Tecnologías Utilizadas
Lenguaje: C++
Paradigma: Programación estructurada

Este proyecto demuestra cómo diferentes estructuras de datos pueden ser utilizadas para resolver problemas reales, mejorando la eficiencia y organización de la información en un sistema de inventario.
